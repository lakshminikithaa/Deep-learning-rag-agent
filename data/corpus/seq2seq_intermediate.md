# Sequence to Sequence (Seq2Seq) Processing

## Encoder-Decoder Architecture
A Sequence to Sequence (Seq2Seq) model is designed to transform an input sequence into an output sequence of a different length. This is achieved using an Encoder-Decoder architecture, typically leveraging Recurrent Neural Networks (RNNs) or LSTMs. The Encoder network processes the input sequence token by token and compresses the entire input information into a single, fixed-length context vector, often called the final hidden state or thought vector. This vector encapsulates the semantic meaning of the entire input sequence. The Decoder network then takes this context vector as its initial state and generates the output sequence token by token until it produces a special end-of-sequence token.

## Information Bottleneck and Attention
A significant limitation of the standard Encoder-Decoder architecture is the information bottleneck problem. Because the Encoder must compress the entire input sequence into a single fixed-length vector, it struggles with very long sequences, often forgetting earlier information. The Attention mechanism was introduced to solve this. Instead of relying solely on the final hidden state, Attention allows the Decoder to "look back" at all the hidden states of the Encoder at every generation step. It computes a dynamic alignment score, creating a custom context vector for each output token, weighting the most relevant input tokens the highest. This dramatically improves performance on complex translation tasks.

## Teacher Forcing in Training
Training a Seq2Seq model can be slow and unstable if the Decoder relies entirely on its own previous predictions, especially early in training when predictions are largely random. To accelerate convergence, a technique called Teacher Forcing is employed. With Teacher Forcing, instead of feeding the model's own generated output from the previous time step as the input to the current time step, the actual ground truth token from the target sequence is supplied. This prevents compounding errors during training. However, it can lead to exposure bias during inference, where the model must rely on its own predictions since ground truth is unavailable.
